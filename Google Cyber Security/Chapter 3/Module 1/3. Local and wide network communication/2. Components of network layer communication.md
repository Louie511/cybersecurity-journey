## 🔷 This lecture focuses on the network layer (Layer 3) of the OSI model, detailing how data packets are addressed and delivered across networks.

## 🔺 Network Layer Operations

- The network layer is responsible for organizing the addressing and delivery of data packets from the source to the destination device.
- Routers use the destination IP address within the packet header to direct packets across the internet, storing this information in routing tables.

## 🔺 IPv4 Packet Format

- An IPv4 packet consists of a header (20-60 bytes) and a data section, with the header containing crucial routing information like source and destination IP addresses.

### ⚫ Key fields in the IPv4 header include: 
- <ins>Version</ins> – Indicates which IP version is used and allows devices to interpret the packet format correctly.
- <ins>Header Length (IHL)</ins> – Specifies the size of the IPv4 header.
- <ins>Type of Service (ToS / DSCP)</ins> – Defines how the packet should be prioritized by routers. ***(Example: Voice traffic may be given higher priority to reduce delay.)***
- <ins>Total Length</ins> – Shows the complete size of the packet, including the header and the data.
- <ins>Identification</ins> – A number used to identify fragments that belong to the same original packet.
- <ins>Flags</ins> – Controls whether the packet can be fragmented during transmission.
- <ins>Fragment Offset</ins> – Indicates the position of a a fragment within the original packet so it can be reassembled correctly.
- <ins>Time To Live (TTL)</ins> – Limits how many routers the packet can pass through before it is discarded.
- <ins>Protocol</ins> – Identifies what type of data or protocol is contained inside the packet.
- <ins>Header Checksum</ins> – Used to verify that the header has not been corrupted during transmission.
- <ins>Source IP Address</ins> – The IP address of the device that sent the packet.
- <ins>Destination IP Address</ins> – The IP address of the device that should receive the packet.
- <ins>Options</ins> – An optional field that allows additional instructions or features to be included in the header.
