## 🔷 This lecture focuses on the network layer (Layer 3) of the OSI model, detailing how data packets are addressed and delivered across networks.

## 🔺 Network Layer Operations

- The network layer is responsible for organizing the addressing and delivery of data packets from the source to the destination device.
- Routers use the destination IP address within the packet header to direct packets across the internet, storing this information in routing tables.

## 🔺 IPv4 Packet Format

- An IPv4 packet consists of a header (20-60 bytes) and a data section, with the header containing crucial routing information like source and destination IP addresses.

### ⚫ Key fields in the IPv4 header include: 
- Version – Indicates which IP version is used and allows devices to interpret the packet format correctly.
- Header Length (IHL) – Specifies the size of the IPv4 header.
- Type of Service (ToS / DSCP) – Defines how the packet should be prioritized by routers. ***(Example: Voice traffic may be given higher priority to reduce delay.)***
- Total Length – Shows the complete size of the packet, including the header and the data.
- Identification – A number used to identify fragments that belong to the same original packet.
- Flags – Controls whether the packet can be fragmented during transmission.
- Fragment Offset – Indicates the position of a a fragment within the original packet so it can be reassembled correctly.
- Time To Live (TTL) – Limits how many routers the packet can pass through before it is discarded.
- Protocol – Identifies what type of data or protocol is contained inside the packet.
- Header Checksum – Used to verify that the header has not been corrupted during transmission.
- Source IP Address – The IP address of the device that sent the packet.
- Destination IP Address – The IP address of the device that should receive the packet.
- Options – An optional field that allows additional instructions or features to be included in the header.
