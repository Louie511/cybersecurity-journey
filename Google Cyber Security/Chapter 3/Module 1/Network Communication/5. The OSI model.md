## 🔷 This lecture provides an in-depth look into the OSI model(open Systems Interconnection), comparing it to the TCP/IP model and detailing the functions of each of its seven layers.

## 🔺 TCP/IP Model vs. OSI Model

- Both models are conceptual frameworks for understanding how data is organized and transmitted across a network.
- The TCP/IP model has four layers, while the OSI model has seven, offering a more detailed view of network processes.

## 🔺 The Seven Layers of the OSI Model

### ⚫ Application Layer ## (Layer 7): 
- This layer directly interacts with the user through applications, handling protocols like HTTP, HTTPS, SMTP, and DNS for internet connectivity.

### ⚫ Presentation Layer (Layer 6): 
- Responsible for data translation, encryption (e.g., SSL), and compression to ensure data is in a standardized format understandable by both sending and receiving systems.

### ⚫ Session Layer (Layer 5): 
- Manages the establishment, maintenance, and termination of connections between devices, including authentication, reconnection, and setting checkpoints for data transfer.

## ⚫ Transport Layer (Layer 4): 
- Focuses on reliable data delivery, segmenting data for transmission, reassembling it at the destination, and managing data flow and speed using protocols like TCP and UDP.

### ⚫ Network Layer (Layer 3): 
- Routes data packets between different networks using IP addresses, ensuring they reach their intended destination.

### ⚫ Data Link Layer (Layer 2): 
- Organizes and manages the sending and receiving of data packets within a single network, involving devices like switches and network interface cards.

### ⚫ Physical Layer (Layer 1): 
- Deals with the physical hardware components of network transmission, such as cables, hubs, and modems, translating data into electrical signals (0s and 1s) for physical transfer.
