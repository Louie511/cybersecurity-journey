## 🔷 This lecture discusses security zones as a network security feature that protects internal networks from external threats.

## 🔺 Network Segmentation and Security Zones

### ⚫ Definition and Purpose: 
    - Security zones are segments of a network with their own access permissions and security rules, acting as barriers to internal networks, maintaining privacy, and preventing issues from spreading.

### ⚫ Real-world Examples: 
  - Examples include a hotel separating guest Wi-Fi from staff networks, and universities dividing networks into faculty and student subnets to isolate potential contamination.

## 🔺 Types of Security Zones

### ⚫ Uncontrolled and Controlled Zones: 
  - The uncontrolled zone refers to external networks like the internet, while the controlled zone is a subnet protecting the internal network from the uncontrolled zone.

### ⚫ Demilitarized Zone (DMZ): 
  - This outer layer within the controlled zone contains public-facing services (e.g., web servers, DNS servers, email servers) that can access the internet, acting as a perimeter to the internal network.
