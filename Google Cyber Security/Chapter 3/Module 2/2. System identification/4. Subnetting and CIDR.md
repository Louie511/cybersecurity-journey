## 🔷 This lecture introduces subnetting as a networking strategy that enhances efficiency and security by dividing a network into smaller, logical groups.

## 🔺 Overview of Subnetting

- Subnetting creates "networks within a network" by subdividing a network address range into smaller subnets.
- This process improves communication efficiency by allowing devices on the same subnet to communicate without involving the broader network, and it can also be used to establish security zones.

## 🔺 Classless Inter-Domain Routing (CIDR)

- CIDR is a method for assigning subnet masks to IP addresses, replacing the older classful addressing system.
- CIDR IP addresses include an IP network prefix (e.g., /24) at the end, which expands the number of available IPv4 addresses and reduces routing table entries.

## 🔺 Security Benefits of Subnetting

- Subnetting allows network professionals to create isolated subnetworks without needing additional IP addresses from their internet service provider.
- This strategy contributes to network security by enabling the creation of isolated subnetworks through physical isolation, routing configurations, and firewalls.
