## 🔷 This lecture provides an overview of network protocols, their categories, and their role in network communication and security.

## 🔺 Overview of Network Protocols

- Network protocols are sets of rules that govern how devices communicate on a network, acting like a common language for data exchange.
- Security analysts need to understand protocols and their vulnerabilities, as malicious actors can exploit them (e.g., DNS for diverting traffic).

## 🔺 Categories of Network Protocols

- Network protocols are broadly categorized into
  1. communication
  2. management
  3. security protocols.
- While there are many protocols, entry-level security analysts should focus on key ones discussed in this reading.

## 🔺 Specific Protocol Examples

### ⚫ Communication Protocols: 
1. Examples include TCP (reliable, connection-oriented) ***TCP occurs at the transport layer***
2. UDP (faster, connectionless) for data transmission ***UDP occurs at the transport layer***
3. HTTP/HTTPS for web communication. ***HTTP uses port 80 and occurs at the application layer***
4. DNS (Domain Name System) Translates domain names into IP addresses so devices can locate servers on the internet. ***Uses UDP port 53, but switches to TCP for large responses. DNS occurs at the application layer***

### ⚫ Management Protocols: 
1. SNMP (Simple Network Management Protocol) – Used to monitor and manage network devices, such as checking bandwidth usage or changing device configurations. ***SNMP occurs at the application layer***
2. ICMP (Internet Control Message Protocol) – Used for error reporting and network diagnostics, commonly used by tools like ping to test connectivity and latency. ***ICMP occurs at the internet layer***

### ⚫ Security Protocols:
1. HTTPS (Hypertext Transfer Protocol Secure) – Secure version of HTTP that uses SSL/TLS encryption to protect web communication. ***Uses port 443 and occurs at the application layer***
2. SFTP (Secure File Transfer Protocol) – Secure protocol for transferring files over a network using SSH encryption. ***typically through TCP port 22 and occurs at the application layer***
