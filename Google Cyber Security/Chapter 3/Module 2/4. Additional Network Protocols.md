## 🔷 This lecture introduces several key networking concepts and protocols essential for a security analyst.

## 🔺 Network Address Translation (NAT)

- NAT allows devices on a private network to communicate with the public internet using a single public IP address.
- It involves a router or firewall replacing private source IP addresses with a public IP address for outgoing messages and performing the reverse for responses. ***NAT is a part of layer 2 (internet layer) and layer 3 (transport layer) of the TCP/IP model.***

## 🔺 IP Address Types

- **Private IP Addresses:** Used for communication within a local network, unique only within that network, and have no cost.
- **Public IP Addresses:** Assigned by ISPs and IANA, unique on the global internet, and incur a leasing cost.

## 🔺 Key Network Protocols

### ⚫ Management Protocols

1. <ins>**DHCP (Dynamic Host Configuration Protocol)**</ins>
Automatically assigns network settings like IP address, DNS server, and default gateway to devices on a network.
- Ports: UDP 67 (server), UDP 68 (client)
- Layer: Application Layer

2. ARP (Address Resolution Protocol)
Translates an IP address into a MAC address so devices can communicate within the same local network.

Port: None (Layer 2 protocol)

Layer: Network Access Layer

Devices store mappings in an ARP cache.
