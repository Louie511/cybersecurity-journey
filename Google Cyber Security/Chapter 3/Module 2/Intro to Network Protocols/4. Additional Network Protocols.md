## 🔷 This lecture introduces several key networking concepts and protocols essential for a security analyst.

## 🔺 Network Address Translation (NAT)

- NAT allows devices on a private network to communicate with the public internet using a single public IP address.
- It involves a router or firewall replacing private source IP addresses with a public IP address for outgoing messages and performing the reverse for responses. ***NAT is a part of layer 2 (internet layer) and layer 3 (transport layer) of the TCP/IP model.***

## 🔺 IP Address Types

- **Private IP Addresses:** Used for communication within a local network, unique only within that network, and have no cost.
- **Public IP Addresses:** Assigned by ISPs and IANA, unique on the global internet, and incur a leasing cost.

## 🔺 Key Network Protocols

### ⚫ Management Protocols

1. <ins>**DHCP (Dynamic Host Configuration Protocol)**</ins> :
  Automatically assigns network settings like IP address, DNS server, and default gateway to devices on a network.
- Ports: UDP 67 (server), UDP 68 (client)
- Layer: Application Layer

2. <ins>**ARP (Address Resolution Protocol)**</ins> :
Translates an IP address into a MAC address so devices can communicate within the same local network.
- Port: None (Layer 2 protocol)
- Layer: Network Access Layer
- Devices store mappings in an ARP cache.

### ⚫ Remote Access Protocols

1. <ins>**Telnet**</ins> :
Allows remote command-line access to another system but sends data in plaintext, making it insecure.
- Port: TCP 23
- Layer: Application Layer

2. <ins>**SSH (Secure Shell)**</ins> :
Secure alternative to Telnet that provides encrypted remote access and authentication.
- Port: TCP 22
- Layer: Application Layer

### ⚫ Email Protocols

1. <ins>**POP3 (Post Office Protocol v3)**</ins> :
Retrieves email by downloading messages from the mail server to a local device.

- Ports:
  - TCP/UDP 110 (unencrypted)
  - TCP/UDP 995 (SSL/TLS encrypted)

- Emails may be removed from the server after download, making multi-device syncing difficult.

2. <ins>**IMAP (Internet Message Access Protocol)**</ins> :
Retrieves email while keeping messages stored on the server, allowing access from multiple devices.
- Ports:
  - TCP 143 (unencrypted)
  - TCP 993 (TLS encrypted)
 
3. <ins>**SMTP (Simple Mail Transfer Protocol)**</ins> :
Used to send and route emails from sender to recipient through mail servers.

- Ports:
  - TCP/UDP 25 (unencrypted)
  - TCP/UDP 587 (TLS encrypted)

- Works with Mail Transfer Agents (MTA) and DNS to deliver email.

### ⚫ Protocols and Port Numbers

- Port numbers tell devices what service or protocol should handle incoming data.
- Firewalls use port numbers to allow or block traffic.
- Security professionals often need to memorize common protocols and their ports for analysis and troubleshooting.
