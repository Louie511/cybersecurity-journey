## 🔷 This lecture explains security hardening as a crucial process for protecting systems from attacks.

## 🔺 Understanding Security Hardening

- Security hardening is the process of strengthening a system to reduce its vulnerability and attack surface.
- The "attack surface" refers to all potential vulnerabilities a threat actor could exploit, similar to doors and windows a robber could use to enter a house.

## 🔺 Methods of Security Hardening

- Regular maintenance procedures, such as software updates (patches) and configuration changes, are performed to keep systems secure.
- Examples include requiring stronger passwords, updating encryption standards, and removing unused applications or services.

## 🔺 Penetration Testing

- Penetration testing (pen testing) is a simulated attack used to identify vulnerabilities in systems, networks, websites, and applications.
- The findings from pen tests help security teams determine and fix security vulnerabilities, strengthening the overall network.
