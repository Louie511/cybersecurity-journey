## 🔷 This lecture focuses on cloud security considerations and introduces the shared responsibility model for security in the cloud.

## 🔺 Cloud Security Challenges

- Cloud computing offers benefits like ease and speed of deployment, cost savings, and scalability, but it also introduces unique security challenges.
- Key challenges include managing identity access, ensuring proper configuration of cloud services, and addressing the increased attack surface that comes with using multiple cloud applications and services.

## 🔺 Zero-Day Attacks and Visibility

- Zero-day attacks, previously unknown exploits, are a concern for both cloud and on-premise networks. Cloud Service Providers (CSPs) are often better equipped to handle these by patching hypervisors and migrating workloads.
- While network administrators have visibility into data packets in both on-premise and cloud networks, CSPs do not allow direct monitoring of traffic on their servers, which can be a concern for some organizations.

## 🔺 Shared Responsibility Model

- The shared responsibility model clarifies that CSPs are responsible for the security of the cloud infrastructure (e.g., physical data centers, hypervisors).
- Organizations using cloud services are responsible for security in the cloud, including their assets, applications, and configurations within the cloud environment.
