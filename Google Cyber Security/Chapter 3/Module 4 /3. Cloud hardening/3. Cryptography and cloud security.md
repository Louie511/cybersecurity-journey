## 🔷 This lecture focuses on securing cloud networks through various hardening practices and the fundamentals of cryptography.

## 🔺 Cloud Security Hardening Techniques

### ⚫ Identity and Access Management (IAM): 
- This involves processes and technologies to manage digital identities and control user access to cloud resources.

### ⚫ Hypervisors: 
- These abstract hardware from the operating software environment, with Type One hypervisors commonly used by cloud service providers (CSPs) who manage their security.

## 🔺 Baselining and Cryptography

### ⚫ Baselining: 
- This involves configuring and setting up a cloud environment with a fixed reference point to compare changes and improve security.

### ⚫ Cryptography in the Cloud: 
- This uses encryption to scramble information into ciphertext, making it unreadable without an encryption key, and is crucial for securing data at rest.

## 🔺 Key Management and Cryptographic Erasure

### ⚫ Cryptographic Erasure: 
- This method, also known as crypto-shredding, involves destroying encryption keys to render encrypted data undecipherable, which is more effective than traditional data destruction in the cloud.

### ⚫ Key Management: 
- Modern encryption relies on securing encryption keys, often using tools like Trusted Platform Modules (TPMs) and Cloud Hardware Security Modules (CloudHSMs) for secure storage and processing.
